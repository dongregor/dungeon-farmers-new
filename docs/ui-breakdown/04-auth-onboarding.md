# Dungeon Farmers - Authentication & Onboarding Flow

**Last Updated:** 2025-12-29
**Status:** Complete flow design based on user stories US-OB-001 to US-OB-027

---

## Overview

This document defines the complete authentication and onboarding flow for Dungeon Farmers, covering:
- Landing page and guest mode
- Account creation and login
- Welcome sequence and first hero
- UI orientation tour
- Tutorial progression
- Returning player experience
- Contextual help systems

### Design Philosophy
- **Zero Friction:** Guest mode enables instant play
- **No FOMO:** Daily rewards have grace periods
- **Respect Player Time:** Tutorials are skippable
- **Lighthearted Tone:** Maintained throughout messaging
- **Progressive Conversion:** Gentle account creation prompts

## 2. Guest vs Account Decision

### Guest Mode Flow
1. User clicks "Play as Guest"
2. Generate temporary session ID
3. Store in `localStorage`
4. Create minimal server-side guest record
5. Redirect to Welcome Sequence

### Guest Session Data
```typescript
interface GuestSession {
  guestId: string           // UUID
  createdAt: Date
  lastActive: Date
  hasSeenTutorial: boolean
  tutorialProgress: number
  // All game data stored locally
}
```

### Conversion Prompts
Display "Create Account" prompt:
- After completing tutorial
- After first Epic+ hero acquired
- After 30 minutes of play
- In settings header (persistent)

**Related Stories:** US-OB-001, US-OB-005
